# §5 Related Work

- **Spivak & Kent (2012), Ologs**: signature + equations, morphisms restricted to functional relations, no implementation in 15 years
- **Goguen & Burstall (1992), Institutions**: signature as core concept, theoretical framework
- **AlgebraicJulia**: category theory for scientific computing, not knowledge management
- **Haskell Functor/Monad**: language-level type classes, not application architecture
- **PKM tools (Obsidian/Logseq/Tana)**: graphs without semantics, edges are not first-class
- **leanblueprint / LeanArchitect**: blueprint ↔ formalization gap, no categorical foundation
- **CompCert / seL4**: verified software, but verified object ≠ verification tool. Astrolabe is both.
