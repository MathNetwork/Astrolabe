# §4 Formal Verification

All definitions and theorems are mechanically verified in Lean 4. The formalization serves as **specification** in two senses:

1. **Algebraic specification** (Goguen & Burstall 1992): Astrolabe signature + axioms define the category
2. **Software specification**: Lean proofs constrain the Python/TypeScript implementation

Two layers of verification:

$$
\texttt{FormalAstrolabeCategory/leancode/} \quad \text{specification — what the code must satisfy (Lean 4)}
$$

$$
\texttt{tests/} \quad \text{engineering — does the code run correctly (pytest + jest)}
$$

They do not replace each other.

## Self-Reference

The formalization is imported back into Astrolabe via $F_{\text{Lean}}$. Opening `FormalAstrolabeCategory/` in Astrolabe shows the dependency graph of its own theory.

This closes a loop: astrolabe manages the formal proofs of its own categorical foundation.
