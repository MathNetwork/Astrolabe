# §3 Concrete Functors

Each functor in the reference implementation is an instance of the general theory. Lean formalization defines the abstract structure; Python/TypeScript implements the computation.

## 3.1 Import Functors

| Functor | Source signature | Lean file |
|---------|-----------------|-----------|
| $F_{\text{Lean}}$ | Lean ConstantInfo + type dependencies | `Functors/LeanImport.lean` |
| $F_{\text{BibTeX}}$ | BibTeX entries + citation relations | (future) |
| $F_{\text{Catala}}$ | Catala legal articles + references | (future) |
| $F_{\text{Python}}$ | Python AST functions/classes + imports | (future) |

Each is a concrete `SignatureMorphism` satisfying the Import Functor theorem.

## 3.2 Enrichment Functors

Enrichment functors add metadata keys without changing objects or morphisms:

$$
A : \mathcal{A}_{\mathcal{M}}(\Sigma) \to \mathcal{A}_{\mathcal{M}'}(\Sigma)
$$

| Functor | Keys added | Lean file |
|---------|------------|-----------|
| Network analysis | betweenness, pagerank, community | `Functors/NetworkAnalysis.lean` |
| Math domain | name, sort, status, statement, proof, notes | `Functors/MathDomain.lean` |
| Timestamp | created_at, updated_at | `Functors/Timestamp.lean` |

## 3.3 Target Functors

Functors from $\mathcal{A}(\Sigma)$ to external target categories.

| Functor | Target category $\mathcal{T}$ | Lean file |
|---------|-------------------------------|-----------|
| Rendering $R$ | $\mathcal{V}$: visual attributes | `Functors/Rendering.lean` |
| Layout $L$ | $\mathcal{P}$: coordinate assignments | `Functors/Layout.lean` |
| Presentation $P$ | $\mathcal{U}_{\text{ui}}$: UI component trees | `Functors/Presentation.lean` |

**Three-layer rendering:** $R_{\text{default}}$ (hash → HSL), $R_{\text{core}}$ (sort → rules), $R_{\text{custom}}$ (user overrides). Overrides are natural transformations:

$$
\eta_{\text{core}} : R_{\text{default}} \Rightarrow R_{\text{core}}, \quad \eta_{\text{custom}} : R_{\text{core}} \Rightarrow R_{\text{custom}}
$$
