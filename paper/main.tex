\begin{abstract}
We introduce Astrolabe, a content-addressable data structure for semantic knowledge management. An astrolabe store is a finite set of entries, each identified by the SHA-256 hash of its content. Each entry carries an ordered reference list (\texttt{ref}) pointing to other entries and an opaque record string. References are unconstrained: entries can reference entries of any degree, yielding a generalized hypergraph structure similar to HyperGraphDB but with content-addressable identity and automatic hash propagation. The store admits two orthogonal decompositions: by degree (reference count) and by stage (reference depth).

To demonstrate the framework, we present MathNetwork, a plugin that bridges informal (\LaTeX) and formal (Lean~4) mathematics by extracting a directed semantic network from the store. In contrast to existing blueprint tools, which label every dependency edge as ``uses'' and nothing more, MathNetwork preserves the semantic nature of each dependency (unfolding, rewriting, specialization, application). This distinction is practically relevant: recent autoformalization systems such as ProofFlow and Aria rely on blueprint-style dependency graphs to guide LLM-driven formalization at scale, yet the coarse annotations available today limit the quality of their output. Astrolabe's richer edge semantics can provide these agents with the context that current blueprints lack.

Astrolabe is implemented as an open-source Tauri desktop application with interactive visualization and an extensible plugin architecture, available at \url{https://github.com/factorin-dev/Astrolabe}.
\end{abstract}
