\begin{abstract}
Knowledge management software universally treats relationships as appendages of nodes---links lack independent identity, type, or content. This limitation renders knowledge networks ineffective for structural analysis. Inspired by category theory, we propose \emph{objects} and \emph{morphisms} as the basic software primitives for knowledge management: both hold equal status, each possessing an independent unique identifier, a typed label (sort), and rich-text content. We rigorously define the \emph{Astrolabe category}---a category freely generated by a knowledge graph, equipped with 12 core morphism types (inclusion, uses, formalizes, equivalent, and others) and an open sort extension mechanism. Building on this model, we implement Astrolabe, a desktop knowledge graph framework that uses MDX as its content substrate and supports data import and network analysis through a plugin system. Focusing on mathematical formalization as the primary application, we demonstrate a closed-loop pipeline: natural language constructs a structured obj/mor network $\to$ network analysis signals guide formalization strategy $\to$ formalization compilation artifacts flow back as standard obj/mor via an import functor $\to$ the network updates and re-analyzes. Astrolabe's design is motivated by a structural gap between blueprint planning and autoformalization in the mathematical formalization community, but the obj/mor model itself is domain-agnostic.
\end{abstract}
