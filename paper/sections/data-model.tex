\section{Data Model}
\label{sec:data-model}

The core data structure of Astrolabe is a content-addressable store of \textbf{entries}, where each entry is a triple of hash, ref, and record. Entries can have arbitrary-length references, forming higher-dimensional semantic structures. The store admits two orthogonal decompositions: by \textbf{degree} (how many references) and by \textbf{stage} (the depth of the reference chain). We describe each in turn.

\subsection{Entry Structure}

An \textbf{astrolabe store} is a finite set of entries. Each entry consists of an ordered reference list pointing to other entries and an opaque record string, with its identity determined by the hash of its content.

\begin{definition}[Content-addressable hypergraph]
\label{def:cah}
Let $H$ be a hash function and let $\Sigma^*$ denote the set of all finite strings over an alphabet $\Sigma$. A \textbf{content-addressable hypergraph} is a finite set $S$ of entries, where each entry $e \in S$ consists of:
\begin{itemize}
  \item an ordered reference list $\mathrm{ref}(e) = (r_1, \ldots, r_k)$ with $r_i \in S$, and
  \item a record string $\mathrm{rec}(e) \in \Sigma^*$,
\end{itemize}
with identity $\mathrm{id}(e) = H(\mathrm{id}(r_1) \| \texttt{0x00} \| \cdots \| \mathrm{id}(r_k) \| \mathrm{rec}(e))$. The \textbf{degree} of $e$ is $|\mathrm{ref}(e)| - 1$. An entry of degree~0 is called an \textbf{atom}.
\end{definition}

The format is general-purpose and can serve any knowledge management scenario by defining appropriate record conventions. In the current implementation, the store is serialized as a single JSON file (\texttt{astrolabe.json}):

\medskip
{\small\ttfamily
\begin{tabular}{@{}l@{}}
\{\\
\ \ \textcolor{gray}{"<12-char-hash>"}:\{\textcolor{blue!70!black}{"ref"}:[\textcolor{gray}{"<hash>"}, ...], \textcolor{red!70!black}{"record"}:\textcolor{gray}{"<string>"}\}\\
\}
\end{tabular}
}
\medskip

\noindent Concretely, each entry has three parts:
\begin{itemize}
  \item \textbf{ref} --- an ordered list of hashes, any length. $|\texttt{ref}|$ defines the \textbf{degree}:
    \begin{itemize}
      \item degree 0: \texttt{ref = [self\_hash]} --- an atom (base unit)
      \item degree 1: \texttt{ref = [A, B]} --- a binary relation
      \item degree $k$: \texttt{ref = [h$_0$, h$_1$, ..., h$_k$]} --- a higher-dimensional semantic relation
    \end{itemize}
  \item \textbf{record} --- a plain string. The core layer does not interpret it; plugins define conventions for structured content (JSON with \texttt{sort}, \texttt{source}, \texttt{title}, \texttt{notes}, etc.).
  \item \textbf{hash} --- $\texttt{SHA256}(\mathit{ref}_1 \| \texttt{0x00} \| \mathit{ref}_2 \| \cdots \| \mathit{record})\texttt{[:12 hex]}$, content-addressable.
\end{itemize}

\paragraph{Hash propagation.}
Because entries are content-addressed, modifying a record changes its hash. When this happens, every entry that references the old hash must be updated and re-hashed recursively. Figure~\ref{fig:propagation} illustrates: editing the record of atom \texttt{a} changes its hash from \texttt{a} to \texttt{a\textsuperscript{$\prime$}}; the edge \texttt{e1} (which references \texttt{a}) is updated to reference \texttt{a\textsuperscript{$\prime$}} and re-hashed to \texttt{e1\textsuperscript{$\prime$}}; then \texttt{e3} (which references \texttt{e1}) is updated and re-hashed to \texttt{e3\textsuperscript{$\prime$}}.

Definition~\ref{def:cah} does not prevent several pathological cases. Consider an existing edge $e_1$:

\medskip
{\small\ttfamily
\begin{tabular}{@{}l@{}}
\textcolor{blue!70!black}{"e1"}:\{"ref":["\textcolor{black}{A}","\textcolor{black}{B}"],"record":"A depends on B"\}
\end{tabular}
}
\medskip

\noindent One could create an entry $e_2$ with $\mathrm{ref}(e_2) = (\mathrm{id}(e_1))$:

\medskip
{\small\ttfamily
\begin{tabular}{@{}l@{}}
\textcolor{gray}{"e2"}:\{"ref":["\textcolor{blue!70!black}{e1}"],"record":"some annotation"\}
\end{tabular}
}
\medskip

\noindent Such an $e_2$ would have degree~0 but would not be a self-contained unit of knowledge; it would be a disconnected wrapper around $e_1$. The correct way to enrich the semantics of $e_1$ is to modify $\mathrm{rec}(e_1)$, which changes $\mathrm{id}(e_1)$ and triggers hash propagation (Figure~\ref{fig:propagation}) to all entries that reference $e_1$. Other pathological cases include dangling references (pointing to entries not in $S$), empty reference lists (yielding degree $-1$), and duplicate elements in a reference list (which would count the same dependency twice). We exclude all such cases with the following definition.

\begin{definition}[Well-formed content-addressable hypergraph]
\label{def:wellformed}
A content-addressable hypergraph $S$ is \textbf{well-formed} if:
\begin{enumerate}
  \item \textbf{(Atom self-reference)} Every atom $e \in S$ satisfies $\mathrm{ref}(e) = (\mathrm{id}(e))$.
  \item \textbf{(Identity uniqueness)} The map $e \mapsto \mathrm{id}(e)$ is injective on $S$; that is, distinct entries have distinct hashes.
  \item \textbf{(Referential closure)} For every $e \in S$ and every $r_i \in \mathrm{ref}(e)$, we have $r_i \in S$.
  \item \textbf{(Non-empty ref)} For every $e \in S$, $|\mathrm{ref}(e)| \geq 1$.
  \item \textbf{(Distinct refs)} For every $e \in S$ with $|\mathrm{ref}(e)| > 1$, the elements of $\mathrm{ref}(e)$ are pairwise distinct.
\end{enumerate}
\end{definition}

\noindent Throughout this paper, all content-addressable hypergraphs are assumed to be well-formed. The implementation enforces these conditions at entry creation time.

\subsection{Stage Decomposition}

Degree measures how many references an entry has, but not \textbf{what those references point to}. Two degree-1 entries can have very different structure: \texttt{e1} references two atoms, while \texttt{f2} references two edges. The \textbf{stage} decomposition captures this depth by classifying entries according to the depth of their reference chains.

Atoms form the base layer. An entry whose refs are all atoms sits one level above; an entry whose refs are all drawn from that level sits one level higher, and so on. We formalize this as follows.

\begin{definition}[Stage decomposition]
Given a well-formed content-addressable hypergraph $S$, define the \textbf{stage filtration} $F_0 \subseteq F_1 \subseteq \cdots \subseteq S$ inductively:
\begin{itemize}
  \item $F_0 = \{ e \in S \mid \deg(e) = 0 \}$ \quad (atoms),
  \item $F_{m+1} = \{ e \in S \mid \forall\, r \in \mathrm{ref}(e),\; r \in F_m \}$.
\end{itemize}
The \textbf{stage} of an entry $e$ is $\mathrm{stage}(e) = \min\{m \mid e \in F_m\}$. In particular, atoms have stage~0, entries whose refs are all atoms have stage~1, and so on. Entries involved in reference cycles belong to no finite stage and are assigned $\mathrm{stage}(e) = -1$.
\end{definition}

\begin{definition}[Reference cycle]
A \textbf{reference cycle} in a content-addressable hypergraph $S$ is a sequence of distinct non-atom entries $e_1, e_2, \ldots, e_n \in S$ ($n \geq 2$) such that $e_{i+1} \in \mathrm{ref}(e_i)$ for $1 \leq i < n$ and $e_1 \in \mathrm{ref}(e_n)$.
\end{definition}

\noindent Reference cycles arise naturally in knowledge management. For example, two logically equivalent theorems may each appear in the proof of the other: an edge ``$A$ proves $B$'' references atom $B$, while an edge ``$B$ proves $A$'' references atom $A$, and a higher-degree entry grouping these two edges into an equivalence relation creates a cycle. Figure~\ref{fig:stage-coloring} shows the same network colored by stage instead of degree, where entries \textcolor{gray}{\texttt{c1}}, \textcolor{gray}{\texttt{c2}}, \textcolor{gray}{\texttt{c3}} form such a cycle and are assigned $\mathrm{stage} = -1$. The key contrast: \textcolor{blue!70!black}{\texttt{e1}} and \textcolor{red!70!black}{\texttt{f2}} both have degree~1, but \textcolor{blue!70!black}{\texttt{e1}} is stage~1 (refs are atoms) while \textcolor{red!70!black}{\texttt{f2}} is stage~2 (refs include stage-1 entries).

\begin{proposition}
For any finite well-formed content-addressable hypergraph $S$, the stage filtration stabilizes: there exists $N$ such that $F_N = F_{N+1} = \cdots$. The set $S \setminus \bigcup_{m \geq 0} F_m$ is exactly the set of entries involved in reference cycles.
\end{proposition}

\begin{proof}
Since $S$ is finite and $F_0 \subseteq F_1 \subseteq \cdots \subseteq S$, the chain must stabilize at some $F_N$.

($\supseteq$) Suppose $e$ belongs to a reference cycle $e_1, \ldots, e_n$. Assume for contradiction that each $e_i$ has finite stage $s_i$. Since $e_{i+1} \in \mathrm{ref}(e_i)$ and $e_i \in F_{s_i}$, all references of $e_i$ lie in $F_{s_i - 1}$, so $\mathrm{stage}(e_{i+1}) < \mathrm{stage}(e_i)$. Following the cycle yields $\mathrm{stage}(e_1) > \mathrm{stage}(e_2) > \cdots > \mathrm{stage}(e_n) > \mathrm{stage}(e_1)$, a contradiction.

($\subseteq$) If $e \notin \bigcup_m F_m$, then $e$ has some reference $r_1 \notin \bigcup_m F_m$ (otherwise $e$ would belong to some $F_m$). Repeating, $r_1$ has a reference $r_2 \notin \bigcup_m F_m$, and so on. Since $S$ is finite, the sequence $e, r_1, r_2, \ldots$ must eventually revisit an entry, forming a reference cycle.
\end{proof}

Reference cycles pose a specific challenge for content-addressable systems that mutable-ID databases such as HyperGraphDB~\cite{HyperGraphDB} do not face. In HyperGraphDB, atoms are identified by database-generated handles; modifying an atom does not change its identity or affect other atoms. In Astrolabe, identity is derived from content: when the record of an entry $e$ changes, $\mathrm{id}(e)$ changes, and every entry referencing $e$ must be updated and re-hashed recursively. If entries form a reference cycle, this propagation could loop indefinitely.

Astrolabe handles this in two ways. First, hash propagation maintains a visited set: when an entry is encountered a second time during propagation, it is skipped, guaranteeing termination. Second, cyclic entries are assigned $\mathrm{stage}(e) = -1$ and excluded from stage-based structural analysis (degree decomposition and plugin computations proceed normally). This approach allows cycles to exist in the store as valid semantic structures while confining their effects on the rest of the system.
