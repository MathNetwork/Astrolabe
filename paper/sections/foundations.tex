\section{The Astrolabe}
\label{sec:foundations}

We build the theory from a single starting point: a finite alphabet. Every externalized piece of knowledge is a finite string; structured knowledge is a finite sequence of key-value pairs of strings. From this minimal assumption we construct hash identifiers, a reference structure, and restriction maps, arriving at the \emph{Astrolabe} $(\Delta(\Sigma), h_{\mathrm{id}})$. Simplices in an Astrolabe are graded by the length of their reference list, yielding the graded set $\Delta(\Sigma) = \bigsqcup_{k \geq 0} \Delta_k$. The restriction maps satisfy $\partial^2 = 0$, giving a chain complex on which the homological algebra of \S\ref{sec:topology} operates unconditionally. The geometric structure (semantic embeddings, metrics) is introduced in \S\ref{sec:semantics}.

\subsection{Alphabet and String Space}

\begin{definition}[Alphabet]
\label{def:alphabet}
An \emph{alphabet} is a finite set $A$. We impose no structure on $A$ beyond finiteness; it may be the set of Unicode code points, ASCII characters, or any other finite encoding.
\end{definition}

\begin{definition}[Monoid]
\label{def:monoid}
A \emph{monoid} is a triple $(M, \cdot, e)$ where $M$ is a set, $\cdot : M \times M \to M$ is an associative binary operation, and $e \in M$ is an identity element: $e \cdot m = m \cdot e = m$ for all $m \in M$.
\end{definition}

\begin{definition}[String space]
\label{def:string-space}
The \emph{string space} $A^*$ is the free monoid on $A$: the set of all finite sequences of elements of $A$, with concatenation as the monoid operation and the empty sequence $\varepsilon$ as identity.
\end{definition}

Every \LaTeX\ file, every Lean source, every JSON document, and every natural-language paragraph is an element of $A^*$ for an appropriate choice of $A$ (e.g., $A_{\mathrm{ASCII}}$ with $|A| = 128$, or the full Unicode code point set).

\begin{theorem}[Universal property of $A^*$]
\label{thm:free-monoid}
For any monoid $(M, \cdot, e)$ and any function $f : A \to M$, there exists a unique monoid homomorphism $\bar{f} : A^* \to M$ extending $f$.
\end{theorem}

In Lean~4, $A^*$ is \texttt{FreeMonoid} in Mathlib, and the universal property is \texttt{FreeMonoid.lift}, a corollary of the adjunction $\mathrm{Free} \dashv U$ between the free monoid functor and the forgetful functor. The universal property guarantees that any structure-preserving interpretation of the alphabet extends uniquely to all strings; this is the algebraic foundation of parsing.

\begin{definition}[Record space]
\label{def:record-space}
A \emph{record} is a finite sequence of key-value pairs, where both keys and values are strings:
\[
  R = (A^* \times A^*)^*.
\]
A record $r = ((k_1, v_1), \ldots, (k_n, v_n)) \in R$ assigns to each key $k_i$ a value $v_i$. Keys may repeat; the order of pairs is significant.
\end{definition}

\subsection{Hash and Content Addressing}

\begin{definition}[Hash space]
\label{def:hash-space}
Let $A_{\mathrm{hex}} = \{0, \ldots, 9, a, \ldots, f\} \subset A$ be the hexadecimal alphabet. The \emph{hash space} is
\[
H = A_{\mathrm{hex}}^*,
\]
the set of finite hexadecimal strings. In particular, $H \subset A^*$.
\end{definition}

Write $H^+ = \bigsqcup_{m \geq 1} H^m$ for the set of non-empty finite ordered tuples of hashes.

\begin{definition}[Content addressing]
\label{def:content-addressing}
A \emph{content-addressing function} is an injective function
\[
h_{\mathrm{id}} : H^+ \times R \to H
\]
that assigns to each pair $(\texttt{ref}, \texttt{record})$ a unique hash. The identity of a simplex is determined by \emph{what it connects} ($\texttt{ref}$) and \emph{what it says} ($\texttt{record}$), not by its location.
\end{definition}

This is the same principle underlying Git commits and IPFS~\cite{Benet2014}. The implementation of $h_{\mathrm{id}}$ (hash function, serialization, collision avoidance) is a system parameter discussed in \S\ref{sec:implementation}; the theory depends only on injectivity.

In practice, $h_{\mathrm{id}}$ is computed by applying a cryptographic hash function (e.g.\ SHA-256) to the serialization of $\texttt{ref}$ and $\texttt{record}$. The implementation details are in \S\ref{sec:implementation}.

\begin{definition}[Parsing]
\label{def:parsing}
Let $\mathcal{P}_{\mathrm{fin}}(R)$ denote the set of finite subsets of $R$:
\[
\mathcal{P}_{\mathrm{fin}}(R) = \{ S \subseteq R \mid |S| < \infty \}.
\]
A \emph{parsing procedure} is a function $P : A^* \to \mathcal{P}_{\mathrm{fin}}(R)$ that maps a string to a finite set of records. Different parsing procedures extract different structures from the same string; we distinguish them by subscripts ($P_{\mathrm{compiler}}$, $P_{\mathrm{LLM}}$, etc.).
\end{definition}

\begin{example}
Consider the Lean source file $s \in A^*$:
\begin{minted}[fontsize=\scriptsize]{lean4}
theorem length_append (l1 l2 : List a) :
    (l1 ++ l2).length = l1.length + l2.length := by
  induction l1 <;> simp [*]
\end{minted}
\noindent Three parsers extract different sets of records from the same string:
\begin{itemize}
\item $P_{\mathrm{compiler}}(s)$: the Lean~4 compiler reads \texttt{.ilean} artifacts and produces one record per declaration. Writing each record as an element of $R = (A^* \times A^*)^*$:
\begin{align*}
r_1 &= ((\texttt{name},\; \texttt{length\_append}),\; (\texttt{sort},\; \texttt{theorem}),\; (\texttt{stmt},\; \ldots\,)) \in R, \\
r_2 &= ((\texttt{name},\; \texttt{List.length}),\; (\texttt{sort},\; \texttt{definition})) \in R, \\
r_3 &= ((\texttt{name},\; \texttt{List.append}),\; (\texttt{sort},\; \texttt{definition})) \in R, \\
r_4 &= ((\texttt{name},\; \texttt{Nat.add}),\; (\texttt{sort},\; \texttt{definition})) \in R.
\end{align*}
Thus $P_{\mathrm{compiler}}(s) = \{r_1, r_2, r_3, r_4\} \in \mathcal{P}_{\mathrm{fin}}(R)$.

\item $P_{\mathrm{LLM}}(s)$: a language model reads the same file as natural language and extracts records for each mathematical concept:
\[
P_{\mathrm{LLM}}(s) = \bigl\{\,
((\texttt{name},\; \texttt{list concatenation}),\; (\texttt{sort},\; \texttt{concept})),\;\ldots
\,\bigr\} \in \mathcal{P}_{\mathrm{fin}}(R).
\]

\item $P_{\mathrm{AST}}(s)$: a syntax-level parser extracts the abstract syntax tree, producing records for each node:
\[
P_{\mathrm{AST}}(s) = \bigl\{\,
((\texttt{name},\; \texttt{length\_append}),\; (\texttt{sort},\; \texttt{theorem})),\;\ldots
\,\bigr\} \in \mathcal{P}_{\mathrm{fin}}(R).
\]
\end{itemize}
\noindent In general $P_{\mathrm{compiler}}(s) \neq P_{\mathrm{LLM}}(s) \neq P_{\mathrm{AST}}(s)$: different parsers extract different records from the same string. The framework is indifferent to domain; the same construction applies to Python modules, legal statutes, or biological databases.
\end{example}

\subsection{Astrolabe File and Stage Decomposition}

\begin{definition}[Astrolabe file]
\label{def:astrolabe-file}
An \emph{astrolabe file} is a finite subset
\[
\mathbb{A} \subset H^+ \times H \times R
\]
satisfying:
\begin{enumerate}
\item (Unique hash) The projection $\pi_H : \mathbb{A} \to H$ taking each entry to its $\texttt{hash}$ is injective.
\item (Atom convention) If $|\texttt{ref}(\sigma)| = 1$, then $\texttt{ref}(\sigma) = [\texttt{hash}(\sigma)]$.
\end{enumerate}
\end{definition}

Condition 2 says: a single-element $\texttt{ref}$ must point to the entry itself. Such an entry is called a \emph{semantic atom}. Entries with $|\texttt{ref}| \geq 2$ are called \emph{semantic forms}. Both are instances of the unified notion of \emph{semantic $k$-form} (Definition~\ref{def:form}).

\begin{definition}[Reference graph]
\label{def:reference-graph}
The \emph{reference graph} of $\mathbb{A}$ is the directed graph $G(\mathbb{A}) = (V, E)$ where $V = \{\texttt{hash}(\sigma) \mid \sigma \in \mathbb{A}\}$ and $(h_1, h_2) \in E$ if and only if $h_2 \in \texttt{ref}(\sigma)$ and $h_2 \neq h_1$, where $\texttt{hash}(\sigma) = h_1$. The self-reference of atoms is excluded.
\end{definition}

\begin{definition}[Cycle]
\label{def:cycle}
A \emph{cycle} in $\mathbb{A}$ is a directed cycle in $G(\mathbb{A})$: a sequence of distinct entries $\sigma_1, \ldots, \sigma_n$ ($n \geq 2$) such that $\sigma_i$ references $\sigma_{i+1}$ for $i = 1, \ldots, n-1$ and $\sigma_n$ references $\sigma_1$. An entry is \emph{cyclic} if it belongs to some cycle.
\end{definition}

With $A$, $A^*$, $R$, $H$, $H^+$, and $h_{\mathrm{id}}$ in hand, we define the combinatorial objects that populate $\mathbb{A}$.

\begin{definition}[Semantic $k$-form]
\label{def:form}
A \emph{semantic $k$-form} ($k \geq 0$) is an entry $\sigma^k \in \mathbb{A}$ with $|\texttt{ref}(\sigma^k)| = k+1$. When $k = 0$, the atom convention in $\mathbb{A}$ forces $\texttt{ref} = [\texttt{hash}]$; we call $\sigma^0$ a \emph{semantic atom}. The set of all semantic $k$-forms in $\mathbb{A}$ is
\[
\mathbb{A}_k = \bigl\{\sigma \in \mathbb{A} \;\big|\; |\texttt{ref}(\sigma)| = k+1\bigr\}.
\]
\end{definition}

Since each entry has a unique $|\texttt{ref}|$, the sets $\mathbb{A}_k$ are pairwise disjoint and
\[
\mathbb{A} = \bigsqcup_{k \geq 0} \mathbb{A}_k.
\]

\begin{remark}[Three decompositions]
\label{rem:orthogonal}
The astrolabe file $\mathbb{A}$ admits three nested decompositions:
\begin{itemize}
\item $\mathbb{A}_k$: by $\texttt{ref}$ \textbf{length} (degree $k = |\texttt{ref}| - 1$).
\item $\mathcal{S}_m$: by $\texttt{ref}$ \textbf{depth} (stage $m$, defined below).
\item $\boldsymbol{\mu}$: by \textbf{multiplicity profile} (which vertices appear and how often, Definition~\ref{def:profile}).
\end{itemize}
The first two are orthogonal: $\mathbb{A}_k$ does not constrain which hashes appear in $\texttt{ref}$, and $\mathcal{S}_m$ does not constrain how many. They intersect in \S\ref{sec:topology} as
\[
\Delta^{(m)}_k = \mathbb{A}_k \cap (\mathcal{S}_{m+1} \setminus \mathcal{S}_m),
\]
which is the set of $k$-simplices at stage $m$. The profile decomposition $\Delta^{(m)}_{\boldsymbol{\mu}}$ refines both: degree and stage are projections of the profile.
\end{remark}

The decomposition $\mathbb{A} = \bigsqcup_k \mathbb{A}_k$ classifies entries by how many hashes their $\texttt{ref}$ contains, but says nothing about \emph{what those hashes point to}. Two 1-forms can have very different internal structure:

{\small\ttfamily
\begin{tabular}{@{}l@{}}
"e1":\{"ref":["a1","a2"]\} \quad -- both refs are atoms\\
"f2":\{"ref":["e1","e2"]\} \quad -- both refs are forms
\end{tabular}
}

\noindent Both belong to $\mathbb{A}_1$, yet \texttt{e1} is a direct relation between atoms while \texttt{f2} is a relation between relations. To run algebraic topology on $\mathbb{A}$, we need restriction maps $d_i$ that land in a well-defined pool of entries; this requires a finer decomposition where, within each layer, every $\texttt{ref}$ draws from the same hash pool. The stage decomposition provides exactly this.

\begin{definition}[Stage decomposition]
\label{def:stage}
The \emph{stages} of $\mathbb{A}$ are defined inductively:
\[
\mathcal{S}_0 = \mathbb{A}_0, \qquad
\mathcal{H}_m = \bigl\{\texttt{hash}(\sigma) \;\big|\; \sigma \in \mathcal{S}_m\bigr\},
\]
\[
\mathcal{S}_{m+1} = \bigl\{\sigma \in \mathbb{A} \;\big|\; \forall h \in \texttt{ref}(\sigma),\; h \in \mathcal{H}_m\bigr\}.
\]
\end{definition}

$\mathcal{S}_0$ is the set of atoms. $\mathcal{H}_m$ is the hash pool at stage $m$. $\mathcal{S}_{m+1}$ is the set of entries in $\mathbb{A}$ whose $\texttt{ref}$ hashes all belong to $\mathcal{H}_m$.

\begin{definition}[Multiplicity profile]\label{def:profile}
Let $\sigma \in \mathcal{S}_{m+1}$ with $\texttt{ref}(\sigma) = [v_0, \ldots, v_k]$, $v_i \in \mathcal{H}_m$.
The \emph{multiplicity profile} of $\sigma$ at stage $m$ is the vector
\[
  \boldsymbol{\mu}(\sigma) = (\mu_1, \ldots, \mu_N) \in \mathbb{Z}_{\geq 0}^N, \qquad
  \mu_j = |\{i : v_i = h_j\}|,
\]
where $\mathcal{H}_m = \{h_1, \ldots, h_N\}$.
The entry $\mu_j$ counts how many times vertex $h_j$ appears in $\texttt{ref}(\sigma)$.
Note $|\boldsymbol{\mu}| = \sum_j \mu_j = k + 1$.
\end{definition}

\begin{definition}[Profile decomposition]\label{def:profile-decomp}
The stage layer $\Delta^{(m)} = \mathcal{S}_{m+1} \setminus \mathcal{S}_m$ decomposes by profile:
\[
  \Delta^{(m)} = \bigsqcup_{\boldsymbol{\mu}} \Delta^{(m)}_{\boldsymbol{\mu}}, \qquad
  \Delta^{(m)}_{\boldsymbol{\mu}} = \{\sigma \in \Delta^{(m)} \mid \boldsymbol{\mu}(\sigma) = \boldsymbol{\mu}\}.
\]
This is the finest decomposition of each stage layer. The two coarser decompositions of \S\ref{sec:foundations} are projections:
\begin{itemize}
\item \emph{Degree:} $\Delta^{(m)}_k = \bigsqcup_{|\boldsymbol{\mu}| = k+1} \Delta^{(m)}_{\boldsymbol{\mu}}$ (forget which vertices, keep count).
\item \emph{Stage:} $\Delta^{(m)} = \bigsqcup_{\boldsymbol{\mu}} \Delta^{(m)}_{\boldsymbol{\mu}}$ (forget profile entirely).
\end{itemize}
\end{definition}

\begin{definition}[Maximal multiplicity]\label{def:max-mult}
The \emph{maximal multiplicity} of $\mathbb{A}$ at stage $m$ is
\[
  m_j^* = \max_{\sigma \in \Delta^{(m)}} \mu_j(\sigma), \qquad
  \mathbf{m}^* = (m_1^*, \ldots, m_N^*).
\]
The profile lattice is
\[
  \mathcal{P}_m = \{\boldsymbol{\mu} \in \mathbb{Z}_{\geq 0}^N : \mu_j \leq m_j^*,\; |\boldsymbol{\mu}| \geq 2\},
\]
with the componentwise partial order $\boldsymbol{\mu} \leq \boldsymbol{\mu}'$ iff $\mu_j \leq \mu'_j$ for all~$j$.
\end{definition}

\begin{proposition}[Profile capacity]\label{prop:profile-capacity}
The number of profile simplices of type $\boldsymbol{\mu}$ is the multinomial coefficient:
\[
  |\mathrm{Simp}(\boldsymbol{\mu})| = \binom{|\boldsymbol{\mu}|}{\boldsymbol{\mu}}
  = \frac{(k+1)!}{\mu_1!\, \mu_2! \cdots \mu_N!}.
\]
The total capacity at stage $m$ under multiplicity bound $\mathbf{m}$ is
\[
  \mathrm{Cap}^{(m)}(\mathbf{m}) = \sum_{\substack{\boldsymbol{\mu} \in \mathcal{P}_m \\ \mu_j \leq m_j}}
  \binom{|\boldsymbol{\mu}|}{\boldsymbol{\mu}}
  = \sum_{L=2}^{\sum_j m_j} L!\cdot[x^L]\prod_{j=1}^{N}
  \sum_{i=0}^{m_j}\frac{x^i}{i!}.
\]
\end{proposition}

\begin{definition}[Profile density]\label{def:profile-density}
The \emph{profile density} of $\mathbb{A}$ at profile $\boldsymbol{\mu}$ is
\[
  \rho_{\boldsymbol{\mu}} = \frac{|\Delta^{(m)}_{\boldsymbol{\mu}}|}{\binom{|\boldsymbol{\mu}|}{\boldsymbol{\mu}}}.
\]
$\rho_{\boldsymbol{\mu}} = 0$: no entry with this profile. $\rho_{\boldsymbol{\mu}} = 1$: saturated.
\end{definition}

\begin{example}[Why profiles matter]\label{ex:why-profiles}
Consider the sample astrolabe file of Figure~\ref{fig:astrolabe-network}, restricted to stage~$0$. The vertex set is $\mathcal{H}_0 = \{\texttt{a1}, \texttt{a2}, \texttt{a3}, \texttt{a4}\}$. Two entries in $\Delta_1^{(0)}$ (both 1-simplices, both degree~1):
\[
  \texttt{e1} = [\texttt{a1}, \texttt{a2}], \qquad
  \texttt{e4} = [\texttt{a4}, \texttt{a1}].
\]
They have the same degree but different profiles: $\boldsymbol{\mu}(\texttt{e1}) = (1, 1, 0, 0)$, $\boldsymbol{\mu}(\texttt{e4}) = (1, 0, 0, 1)$. The degree decomposition $\Delta_1^{(0)}$ puts them together; the profile decomposition $\Delta^{(0)}_{\boldsymbol{\mu}}$ separates them.

Now consider a hypothetical entry with a repeated vertex:
\[
  \sigma = [\texttt{a1}, \texttt{a1}, \texttt{a2}], \qquad
  \boldsymbol{\mu}(\sigma) = (2, 1, 0, 0).
\]
This records that \texttt{a1} plays \emph{two distinct roles} in a ternary relation with \texttt{a2}; for instance, a theorem that uses the same definition twice in different ways. In a Lean~4 formalization, this occurs when a proof applies the same lemma to two different subterms.

Three structural facts are visible at the profile level but invisible at the degree level:
\begin{enumerate}
\item $\sigma$ and $\texttt{f1} = [\texttt{a1}, \texttt{a2}, \texttt{a3}]$ are both 2-simplices ($k = 2$), but $\boldsymbol{\mu}(\sigma) = (2, 1, 0, 0)$ while $\boldsymbol{\mu}(\texttt{f1}) = (1, 1, 1, 0)$. The degree decomposition cannot distinguish them; the profile decomposition can.
\item The profile boundary $\partial^{(\texttt{a1})}(\sigma) = [\texttt{a1}, \texttt{a2}] - [\texttt{a1}, \texttt{a2}] = 0$: the two copies of \texttt{a1} cancel. But $\partial^{(\texttt{a1})}(\texttt{f1}) = [\texttt{a2}, \texttt{a3}] \neq 0$. The total boundary $\partial$ mixes these; the profile boundary $\partial^{(j)}$ separates them.
\item The profile density $\rho_{(2,1,0,0)}$ measures what fraction of ``\texttt{a1}-repeated-twice-with-\texttt{a2}'' relations actually appear. If $\rho_{(2,1,0,0)} = 0$ but $\rho_{(1,1,1,0)} = 1$, the knowledge base has no repeated-vertex relations: it is a classical simplicial set. If $\rho_{(2,1,0,0)} > 0$, the knowledge base genuinely uses repeated references.
\end{enumerate}
\end{example}

\begin{lemma}[Stage exhaustion]
\label{lem:stage-exhaustion}
Every entry in $\mathbb{A}$ belongs to some stage:
\[
\mathbb{A} = \bigcup_{m \geq 0} \mathcal{S}_m.
\]
\end{lemma}

\begin{proof}
An entry with $\texttt{ref} = [\texttt{hash}]$ belongs to $\mathcal{S}_0$. Otherwise, every hash in $\texttt{ref}$ points to an entry in some $\mathcal{S}_m$; taking the maximum such $m$ places the entry in $\mathcal{S}_{m+1}$.
\end{proof}

\begin{lemma}[Cyclic entries have infinite stage]
\label{lem:cyclic-infinite}
If $\sigma$ is cyclic, then $\operatorname{stage}(\sigma) = \infty$.
\end{lemma}

\begin{proof}
Let $\sigma_1 \to \sigma_2 \to \cdots \to \sigma_n \to \sigma_1$ be a cycle. We show by induction that $\sigma_i \notin \mathcal{S}_m$ for any finite $m$ and any $i$.

\emph{Base case.} No $\sigma_i$ is an atom: an atom has $\texttt{ref} = [\texttt{hash}]$, but each $\sigma_i$ references $\sigma_{i+1} \neq \sigma_i$. Hence $\sigma_i \notin \mathcal{S}_0$.

\emph{Inductive step.} Suppose $\sigma_i \notin \mathcal{S}_m$ for all $i$. Then $\texttt{hash}(\sigma_i) \notin \mathcal{H}_m$ for all $i$. Each $\sigma_i$ references $\sigma_{i+1}$, so $\texttt{ref}(\sigma_i)$ contains $\texttt{hash}(\sigma_{i+1}) \notin \mathcal{H}_m$. Hence $\sigma_i \notin \mathcal{S}_{m+1}$.

By induction, $\sigma_i \notin \mathcal{S}_m$ for all $m$, so $\operatorname{stage}(\sigma_i) = \infty$.
\end{proof}

\begin{definition}[Reference depth]
\label{def:depth}
The \emph{reference depth} of $\sigma \in \mathbb{A}$ is
\[
\operatorname{depth}(\sigma) = \begin{cases} 0 & \text{if } \sigma \text{ is an atom,} \\ 1 + \max\{\operatorname{depth}(\tau) : \texttt{hash}(\tau) \in \texttt{ref}(\sigma),\; \tau \neq \sigma\} & \text{if } \sigma \text{ is acyclic,} \\ \infty & \text{if } \sigma \text{ is cyclic.} \end{cases}
\]
\end{definition}

For acyclic entries, $\operatorname{depth}(\sigma) = \operatorname{stage}(\sigma)$.

\begin{theorem}[Cycle--stage independence]
\label{thm:cycle-stage}
Let $\mathbb{A}^{\circ} = \{\sigma \in \mathbb{A} \mid \operatorname{stage}(\sigma) < \infty\}$ be the acyclic part and $\mathbb{A}^{\circlearrowleft} = \{\sigma \in \mathbb{A} \mid \operatorname{stage}(\sigma) = \infty\}$ the cyclic part. Then:
\begin{enumerate}
\item The stage decomposition of $\mathbb{A}^{\circ}$ is independent of $\mathbb{A}^{\circlearrowleft}$: removing or adding cyclic entries does not change $\mathcal{S}_m$ for any finite $m$.
\item No acyclic entry references a cyclic entry.
\item $\mathcal{S}_m \subseteq \mathbb{A}^{\circ}$ for all $m$.
\end{enumerate}
\end{theorem}

\begin{proof} \mbox{}
\begin{enumerate}
\item $\mathcal{S}_0 = \mathbb{A}_0$ consists of atoms, which are acyclic. $\mathcal{S}_{m+1}$ depends only on $\mathcal{H}_m = \{\texttt{hash}(\sigma) \mid \sigma \in \mathcal{S}_m\}$. By induction $\mathcal{S}_m \subseteq \mathbb{A}^{\circ}$, so cyclic entries never enter $\mathcal{S}_m$ and never contribute to $\mathcal{H}_m$.
\item Suppose acyclic $\sigma$ references cyclic $\tau$. Then $\texttt{hash}(\tau) \notin \mathcal{H}_m$ for any $m$, so $\sigma \notin \mathcal{S}_{m+1}$ for any $m$, contradicting $\operatorname{stage}(\sigma) < \infty$.
\item Immediate from (2).
\end{enumerate}
\end{proof}

\begin{remark}[Cycle-profile independence]
Theorem~\ref{thm:cycle-stage} extends to profiles: cyclic entries have no finite profile (they belong to no $\Delta^{(m)}_{\boldsymbol{\mu}}$ for any finite $m$ or $\boldsymbol{\mu}$), and removing or adding cyclic entries does not change $\Delta^{(m)}_{\boldsymbol{\mu}}$ for any $\boldsymbol{\mu}$.
\end{remark}

The cyclic part $\mathbb{A}^{\circlearrowleft}$ lives outside the stage tower entirely. The algebraic topology of \S\ref{sec:topology}, defined on stage layers $\Delta^{(m)} = \mathcal{S}_{m+1} \setminus \mathcal{S}_m$, sees only the acyclic part. Cyclic entries are valid members of $\mathbb{A}$ but do not participate in the stage decomposition or its algebraic structure.

\begin{proposition}[Same-stage characterization]
\label{prop:same-stage}
An entry $\sigma \in \mathcal{S}_{m+1} \setminus \mathcal{S}_m$ if and only if
\[
\max_{h \in \texttt{ref}(\sigma)} \operatorname{stage}(h) = m.
\]
That is, entries in the same stage share the same maximum reference depth.
\end{proposition}

\begin{proof}
If $\sigma \in \mathcal{S}_{m+1} \setminus \mathcal{S}_m$, then every hash in $\texttt{ref}(\sigma)$ points to $\mathcal{S}_m$, so all reference stages are $\leq m$. If all were $\leq m-1$, then $\sigma$ would belong to $\mathcal{S}_m$, contradicting $\sigma \notin \mathcal{S}_m$. Hence the maximum is exactly $m$. The converse is immediate from the definition.
\end{proof}

\begin{proposition}[No intra-stage references]
\label{prop:no-intra-stage}
No entry in $\mathcal{S}_{m+1} \setminus \mathcal{S}_m$ references another entry in $\mathcal{S}_{m+1} \setminus \mathcal{S}_m$.
\end{proposition}

\begin{proof}
If $\sigma, \tau \in \mathcal{S}_{m+1} \setminus \mathcal{S}_m$ and $\sigma$ references $\tau$, then $\operatorname{stage}(\tau) \leq m$ by definition of $\mathcal{S}_{m+1}$. But $\tau \notin \mathcal{S}_m$, contradiction. (No acyclicity assumption is needed: by Theorem~\ref{thm:cycle-stage}, all entries in $\mathcal{S}_{m+1}$ are acyclic.)
\end{proof}

\begin{proposition}[Uniform vertex pool]
\label{prop:uniform-pool}
For every entry $\sigma \in \mathcal{S}_{m+1} \setminus \mathcal{S}_m$, all hashes in $\texttt{ref}(\sigma)$ belong to $\mathcal{H}_m$. In particular, $\texttt{ref}(\sigma) \in \mathcal{H}_m^{k+1}$ where $k = |\texttt{ref}(\sigma)| - 1$.
\end{proposition}

This means: within each stage layer $\mathcal{S}_{m+1} \setminus \mathcal{S}_m$, every entry draws its references from the same hash pool $\mathcal{H}_m$. The $\texttt{ref}$ length $k+1$ then plays the role of a uniform dimension within this pool.

These properties make each stage layer a natural home for algebraic topology. The profile decomposition (Definition~\ref{def:profile-decomp}) refines this further. Within $\Delta^{(m)}_{\boldsymbol{\mu}}$:

\begin{itemize}
\item The vertex set is $\mathcal{H}_m$, and the exact usage pattern is fixed by $\boldsymbol{\mu}$: vertex $h_j$ appears exactly $\mu_j$ times in every ref.
\item The ref length $|\boldsymbol{\mu}|$ is the dimension. Entries in $\Delta^{(m)}_{\boldsymbol{\mu}}$ differ only in the \emph{ordering} of their vertices, not in which vertices appear or how often.
\item No entry in $\Delta^{(m)}_{\boldsymbol{\mu}}$ references any entry in $\Delta^{(m)}_{\boldsymbol{\mu}'}$ (immediate from Proposition~\ref{prop:no-intra-stage}).
\item The number of possible orderings is the multinomial coefficient $\binom{|\boldsymbol{\mu}|}{\boldsymbol{\mu}}$ (Proposition~\ref{prop:profile-capacity}). The profile density $\rho_{\boldsymbol{\mu}}$ (Definition~\ref{def:profile-density}) measures what fraction of these orderings actually appears in $\mathbb{A}$.
\end{itemize}

In \S\ref{sec:topology}, we exploit this structure at two levels. At the \emph{coarse level}, we define the boundary operator $\partial$ on each stage layer, verify $\partial^2 = 0$, and define homology groups $H_k(\Delta^{(m)})$. At the \emph{profile level}, $\partial$ decomposes into $N$ operators $\partial^{(1)}, \ldots, \partial^{(N)}$ (one per vertex) satisfying $\partial^{(j)} \circ \partial^{(l)} + \partial^{(l)} \circ \partial^{(j)} = 0$ (anticommutativity). This makes each stage layer a $\mathbb{Z}^N$-graded multicomplex, with $N$ independent directions of homology $H^{(j)}_{\boldsymbol{\mu}}$ and multigraded Betti numbers $\beta^{(j)}_{\boldsymbol{\mu}}$ that refine the classical $\beta_k$. The coarse homology is the projection $H_k = \bigoplus_{|\boldsymbol{\mu}|=k+1} H_{\boldsymbol{\mu}}$ under the total boundary $\partial = \sum_j \partial^{(j)}$.
